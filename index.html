Antes de Programar

Entendamos algunos conceptos claves!.

Por Melania La Rosa

¿Qué significa programar? -----> Abstraer

Significa tomar escenarios de la vida real y representarlos de una manera que la computadora puede entender y ejecutar. Entonces es vital adquirir esta destreza de abstraer para resolver problemas de manera eficiente y efectiva en el desarrollo de software.

¿Cómo debo pensar para Abstraer? -----> Pensamiento Computacional

Es un enfoque para resolver problemas de manera estructurada y lógica, dividiendo el problema en partes más pequeñas y manejables, y diseñando soluciones que pueden ser ejecutadas por una máquina. El pensamiento computacional se aplica en cualquier tipo de desarrollo de software o programación. Se trata de pensar como un programador, dividiendo problemas complejos en componentes simples, identificando patrones y desarrollando soluciones paso a paso antes de programar. Los pasos son los siguientes: (1)Descomposición, (2)Reconocimiento de Patrones, (3)Abstracción y (4)Diseño de Algoritmos Veamos algunos ejemplos!

Ejemplo 1: Preparar una Taza de Café

Vamos a imaginar que vamos a escribir un programa que simule cómo preparar una taza de café

  1. Descomposición: Divide el problema en tareas más pequeñas.
    • Calentar el agua.
    • Preparar el filtro.
    • Colocar el café en el filtro.
    • Verter el agua caliente sobre el café.
    • Esperar a que el café se filtre.
    • Servir el café en una taza.
  2. Reconocimiento de Patrones: Identifica las partes que se repiten.
    • Tal vez reconoces que cada vez que preparas café, primero necesitas calentar el agua y preparar el filtro, sin importar si es café o té.
  3. Abstracción: Ignoras detalles innecesarios.
    • Tal vez reconoces que cada vez que preparas café, primero necesitas calentar el agua y preparar el filtro, sin importar si es café o té.
  4. Diseño de Algoritmos: Crea un conjunto de pasos ordenados para resolver el problema. Escribes un conjunto de instrucciones que la computadora puede seguir.Esto es popularmente llamado "Pseudocódigo"

    Pseudocódigo:

    Inicio

    • Llenar una tetera con agua.
    • Calentar la tetera hasta que el agua hierva.
    • Colocar el filtro en la cafetera.
    • Añadir 2 cucharadas de café molido al filtro.
    • Verter el agua caliente sobre el café en el filtro.
    • Esperar 5 minutos para que el café se filtre.
    • Servir el café en la taza.

    Fin

  5. Programar: Ahora si vamos a programar !

    Ya lo podemos programar simbólicamente en cualquier lenguaje de programación! , lo haremos en JavaScript tomando en cuenta lo que ya hemos visto en las clases:

    function prepararCafe() {

    console.log("Llenar tetera con agua.");

    console.log("Calentar el agua hasta que hierva.");

    console.log("Colocar filtro en la cafetera.");

    console.log("Verter agua caliente sobre el café.");

    ¿Te diste cuenta que antes de llegar al paso 5, es necesario hacer los otros pasos previos?

    Con el tiempo y la dedicación podrás obtener la habilidad de abstraer y los primeros pasos los harás de forma automática, es decir podrás llegar al paso 5 de forma rápida

Ejemplo 2: Calculadora Básica

Este es un ejemplo de tecnología, ya es mas cercano a lo que haremos para programar

  1. Descomposición: Divide el problema en tareas más pequeñas.
    • Pedir al usuario que ingrese el primer número.
    • Pedir al usuario que ingrese el operador (suma, resta, multiplicación o división).
    • Pedir al usuario que ingrese el segundo número.
    • Realizar la operación matemática correspondiente.
    • Mostrar el resultado al usuario.
  2. Reconocimiento de Patrones: Identifica las partes que se repiten.
    • Tomar dos números y un operador.
    • Realizar la operación solicitada.
    • Mostrar el resultado al usuario.
  3. Abstracción: Ignoras detalles innecesarios.
    • No nos interesa cómo se realiza internamente cada operación (ya que JavaScript puede hacerlo por nosotros con los operadores matemáticos estándar). Nos concentramos en la entrada del usuario y la ejecución de la operación.
  4. Diseño de Algoritmos: Crea un conjunto de pasos ordenados para resolver el problema.

    Pseudocódigo:

    Inicio

    • Solicitar al usuario el primer número.
    • Solicitar al usuario el operador (+, -, *, /).
    • Solicitar al usuario el segundo número.
    • Si el operador es "+" entonces sumar los números.
    • Si el operador es "-" entonces restar los números.
    • Si el operador es "*" entonces multiplicar los números.
    • Si el operador es "/" entonces dividir los números.
    • Mostrar el resultado.

    Fin

  5. Programar: Ahora si vamos a programar !

    Ya lo podemos programar simbólicamente en cualquier lenguaje de programación!, lo haremos en JavaScript tomando en cuenta lo que ya hemos visto en las clases:

    function calculadora() {

    // Paso 1: Pedir el primer número

    const numero1 = parseFloat(prompt("Ingresa el primer número:"));

    // Paso 2: Pedir el operador

    const operador = prompt("Ingresa el operador (+, -, *, /):");

    // Paso 3: Pedir el segundo número

    const numero2 = parseFloat(prompt("Ingresa el segundo número:"));

    let resultado;

    // Paso 4: Realizar la operación

    if (operador === "+") {

    resultado = numero1 + numero2;

    } else if (operador === "-") {

    resultado = numero1 - numero2;

    } else if (operador === "*") {

    resultado = numero1 * numero2

    } else if (operador === "/") {

    resultado = numero1 / numero2;

    } else {

    alert("Operador inválido");

    return;

    }

    // Paso 5: Mostrar el resultado

    alert(`El resultado es: ${resultado}`);

    }

    // Llamamos a la función calculadora

    calculadora();
